Dateistruktur

Nutze das Atomic Design Pattern, um deine CSS-Dateien zu strukturieren. Unsere Interpretation lautet wie folgt:

  1. Settings: Definition von Farben, Größen usw.
  2. Mixins: Eigene SCSS-Mixins
  3. Utilities: Klassen für schnelles layouten
  4. Atoms: HTML-Elemente wie bspw. Buttons und Links
  5. Molecules: UI-Elemente, welche aus Atoms bestehen
  6. Modules: UI-Elemente, welche aus Molecules bestehen
  7. Templates: Wiederverwendbare Layouts
  8. Pages: Seitenspezifische Layouts

Wenn externe CSS-Ressourcen wie bspw. die normalize.css eingesetzt werden, dann werden diese im Ordner 0-vendor gespeichert.

Innerhalb jedes Ordners werden die einzelnen SCSS-Dateien mittels einer eigenen _-section-index.scss importiert. Im Hauptverzeichnis importiert die main.scss alle einzelnen Index-Dateien.

{% highlight text %} scss/ ├── 0-vendor | ├── _-vendor-index.scss | └── _normalize.css ├── 1-settings | ├── _-setrings-index.scss | ├── _colors.scss | └── _fonts.scss ├── 2-mixins ├── 3-utilities ├── 4-atoms ├── 5-molecules ├── 6-modules ├── 7-templates ├── 8-pages └── main.scss {% endhighlight %}
