Dateistruktur

Nutze das Atomic Design Pattern, um deine CSS-Dateien zu strukturieren. Unsere Interpretation lautet wie folgt:

  1. Settings: Definition von Farben, Größen usw.
  2. Mixins: Eigene SCSS-Mixins
  3. Utilities: Klassen für schnelles layouten
  4. Atoms: Kleinstmögliche UI-Elemente, wie bspw. Buttons, Pills, Dropdowns, …
  5. Molecules: UI-Elemente, welche aus Atoms bestehen
  6. Organisms: UI-Elemente, welche aus Molecules bestehen
  7. Templates: Wiederverwendbare Layouts
  8. Pages: Seitenspezifische Layouts

Wenn externe CSS-Ressourcen wie bspw. die normalize.css eingesetzt werden, dann werden diese im Ordner 0-vendor gespeichert.

Innerhalb jedes Ordners werden die einzelnen SCSS-Dateien mittels eines eigenen Bundles, z.B. __bundle.scss, importiert. Im Hauptverzeichnis importiert die Haupt-SCSS-Datei, bspw. application.scss, alle einzelnen Bundle-Dateien.
