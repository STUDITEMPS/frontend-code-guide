Syntax

  • Benutze Soft-Tabs mit 2 Leerzeichen – es ist die einzige Möglichkeit zu garantieren, dass der Code in allen Umgebungen gleich gerendert wird.
  • Lasse beim gruppieren von Selektoren jedem Selektor seine eigene Zeile.
  • In Fällen, in denen ein Regelsatz nur eine einzige Deklaration enthält, kann dieser in einer Zeile mit dem Selektor stehen.
  • Für bessere Lesbarkeit, füge ein Leerzeichen vor jeder öffnenden Klammer eines Deklarationsblocks hinzu.
  • Platziere schließende Klammern in einer neuen Zeile.
  • Füge ein Leerzeichen hinter jedem : für jede Deklaration hinzu.
  • Für ein genaueres Fehlerreporting sollte jede Deklaration in einer eigenen Zeile stehen.
  • Beende jede Deklaration mit einem Semikolon. Für die letzte Deklaration ist dies optional.
  • Kommagetrennte Eigenschaftswerte sollten ein Leerzeichen nach jedem Komma enthalten (z.B.box-shadow).
  • Füge Eigenschaftswerten oder Farbwerten keine führenden Nullen hinzu (z.B., .5 anstatt 0.5 und -.5px anstatt -0.5px).
  • Schreibe Hexwerte klein, z.B. #fff. Kleingeschriebene Buchstaben sind beim Lesen eines Dokumentes einfacher zu unterscheiden, da sie mehr einzigartige Formen aufweisen.
  • Verwende kurze Hexwerte, wenn möglich. Z.B. #fff anstatt #ffffff.
  • Setze Attributwerte in Anführungszeichen, z.B. input[type='text']. Sie sind nur in einigen Fällen optional und es ist eine Gewöhnung an Konsistenz.
  • Vermeide das Setzen von Einheiten für Nullwerte, z.B. margin: 0; anstatt margin: 0px;.
