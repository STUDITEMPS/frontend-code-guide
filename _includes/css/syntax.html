CSS Syntax

  • Benutze „Soft-Tabs” mit 2 Leerzeichen um zu garantieren, dass der Code in allen Umgebungen gleich gerendert wird.
  • Verwende beim Gruppieren von Selektoren für jeden Selektor eine eigene Zeile.
  • In Fällen, in denen ein Regelsatz nur eine einzige Deklaration enthält, kann dieser in einer Zeile mit dem Selektor stehen.
  • Für bessere Lesbarkeit, füge ein Leerzeichen vor jeder öffnenden Klammer eines Deklarationsblocks hinzu.
  • Platziere schließende Klammern in eine neue Zeile.
  • Füge ein Leerzeichen hinter jedem : einer Deklaration hinzu.
  • Für ein genaueres Fehlerreporting sollte jede Deklaration in einer eigenen Zeile stehen.
  • Beende jede Deklaration mit einem Semikolon. Für die letzte Deklaration ist dieses optional.
  • Kommagetrennte Eigenschaftswerte sollten ein Leerzeichen nach jedem Komma enthalten (z.B.box-shadow).
  • Füge Eigenschaftswerten oder Farbwerten keine führenden Nullen hinzu (z.B., .5 anstatt 0.5 und -.5px anstatt -0.5px).
  • Schreibe Hexwerte klein, z.B. #fff. Kleingeschriebene Buchstaben sind beim Lesen eines Dokumentes einfacher zu unterscheiden, da sie einzigartigere Formen aufweisen.
  • Verwende kurze Hexwerte, wenn möglich. Z.B. #fff anstatt #ffffff.
  • Setze Attributwerte in Anführungszeichen, z.B. input[type='text']. In einigen Fällen sind diese optional, führen aber zu einem konsistenten Code.
  • Vermeide das Setzen von Einheiten für Nullwerte, z.B. margin: 0; anstatt margin: 0px;.
{% highlight css %} /* Schlechtes CSS */ .selector, .selector-secondary, .selector[type="text"] { padding:15px; margin:0px 0px 15px; background-color:rgba(0, 0, 0, 0.5); box-shadow:0px 1px 2px #CCC,inset 0 1px 0 #FFFFFF } .span1 { width: 60px; } /* Gutes CSS */ .selector, .selector-secondary, .selector[type='text'] { padding: 15px; margin-bottom: 15px; background-color: rgba(0,0,0,.5); box-shadow: 0 1px 2px #ccc, inset 0 1px 0 #fff; } .span1 { width: 60px } {% endhighlight %}
